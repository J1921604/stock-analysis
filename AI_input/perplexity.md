
## 電力会社経営状況ダッシュボード仕様書

### **主要セクション**

**1. システムアーキテクチャ**

- GitHub を中心とした軽量インフラ設計
- GitHub Actions による日次自動バッチ処理(毎日 18:00 JST)
- SQLite ファイルによる一元管理
- GitHub Pages でのブラウザ解析ページ提供

**2. データベース設計（7つのテーブル）**

- 企業マスタ、決算情報、貸借対照表
- 主要経営指標、株価データ、セグメント情報
- データソース追跡テーブル
- 完全な SQL スキーマと索引設計

**3. 推奨経営指標（39種類以上）**

- **業績関連**: 経常利益（期ずれ除き）、EBITDA、フリー・キャッシュ・フロー
- **安全性**: 自己資本比率、D/Eレシオ、ネット・デット/EBITDA
- **収益性**: ROE、ROA、配当利回り、EPS
- **電力事業特有**: 販売電力量、契約件数、平均売電単価、原子力稼働率

**4. データ取得ソース**

- EDINET API（金融庁・XBRL形式）
- TDnet API（東証・適時開示）
- 各社 IR サイト
- Yahoo! ファイナンス（株価）

**5. ブラウザ上での解析**

- HTML/JavaScript による動的ダッシュボード
- SQLite3.js でオフライン完全分析
- ローカルファイル指定対応
- Presigned URL による自動ロード

**6. 通知・アラート仕様**

- GitHub Issues による日次自動通知
- ROE、配当性向、債務比率などの異常値検知
- 通知ルール（severity レベル付き）

**7. 実装詳細**

- GitHub Actions の YAML 設定
- Python スクリプト（fetch、parse、update、notify）
- JavaScript アプリケーション（カード生成、グラフ化）
- 11 個のサンプル実装コード

**8. 推奨カード・ウィジェット**

- 財務サマリー、安全性指標、販売電力量推移
- セグメント別損益、3社比較、KPI トレンド

**9. 導入ガイド**

- ローカル開発環境構築手順
- GitHub Actions デプロイ設定
- トラブルシューティング
- 今後の拡張候補機能
